# Travel App — Phase 1 Design Spec
**Date:** 2026-03-11
**Status:** Approved

---

## Overview

A self-service travel planning consumer app that allows users to plan their own trips without needing a travel agency or professional planner. The target audience is general — any traveler, any age. The design direction is **Warm & Adventurous**: bold oranges, earthy tones, and a friendly yet modern aesthetic.

Phase 1 establishes the project foundation: tooling setup, design system, and a login screen as the first functional UI. The login form is **UI-only** — no authentication logic is implemented in Phase 1.

## Setup Steps

1. `pnpm create vite@latest . -- --template react-ts` (scaffold into current dir)
2. `pnpm install`
3. `pnpm install tailwindcss@3 postcss autoprefixer`
4. `npx tailwindcss init -p` — generates `tailwind.config.js` (rename to `.js`, not `.ts`, to avoid ts-node requirement)
5. `pnpm dlx shadcn-ui@latest init` — uses the latest 0.x release (confirmed Tailwind v3 compatible). Generates `components.json`, `src/lib/utils.ts`, updates `tailwind.config.js` and `src/index.css`
6. `pnpm install react-router-dom @fontsource/inter react-icons lucide-react`
7. **After shadcn init**, edit `tailwind.config.js` to add design tokens (colors via `var()` references, fontFamily) — editing before step 5 risks shadcn init overwriting the file
8. **After shadcn init**, append CSS custom properties to the `:root` block in `src/index.css` — add below shadcn's generated variables, do not replace them
9. Import Inter font weights in `src/main.tsx` (first lines, before any component imports)

> **Note:** Use `tailwind.config.js`, not `.ts`. The `.ts` extension requires ts-node/tsx to be configured and is not supported out of the box with Vite + Tailwind v3.

## Design System

### Colors — Tailwind Config + CSS Variables

Define in `tailwind.config.js` under `theme.extend.colors` and as CSS custom properties in `src/index.css` for use with shadcn/ui:

```css
/* src/index.css */
:root {
  --primary: #FF6B35;
  --primary-dark: #C75B2A;
  --primary-light: #F7C59F;
  --background: #FEFAF6;
  --foreground: #1A0A00;
  --muted: #78716C;
  --input-bg: #F5F0EB;
  --border: #E5E0DB;
}
```

```js
// tailwind.config.js — reference CSS variables so both systems stay in sync
colors: {
  primary: { DEFAULT: 'var(--primary)', dark: 'var(--primary-dark)', light: 'var(--primary-light)' },
  background: 'var(--background)',
  foreground: 'var(--foreground)',
  muted: 'var(--muted)',
  'input-bg': 'var(--input-bg)',
  border: 'var(--border)',
}
```

> Tailwind color utilities (e.g. `bg-primary`, `text-muted`) resolve via CSS variables. This keeps the token system in sync — change a value in `:root` and it updates everywhere.

| Token | Hex | Usage |
|-------|-----|-------|
| Primary | `#FF6B35` | CTA buttons, links, accents |
| Primary Dark | `#C75B2A` | Hover states, gradients |
| Primary Light | `#F7C59F` | Backgrounds, gradients |
| Background | `#FEFAF6` | App background |
| Foreground | `#1A0A00` | Headings, body text |
| Muted | `#78716C` | Secondary text, labels, hints |
| Input BG | `#F5F0EB` | Form input backgrounds |
| Border | `#E5E0DB` | Dividers, input borders at rest |

| Heading L | 22px | 800 | |
| Heading M | 20px | 700 | |

| Small | 13px | 400 | Color: Muted |

### Spacing & Shape

- Card border radius: `rounded-2xl` (16px)
- Button/Input border radius: `rounded-xl` (12px)
- Card shadow: `shadow-[0_8px_40px_rgba(0,0,0,0.18)]`
- Button shadow (primary): `shadow-[0_4px_14px_rgba(255,107,53,0.4)]`

### Components

- **Primary Button:** bg `primary`, white text, `rounded-xl`, primary shadow, hover: bg `primary-dark`
- **Outline Button:** transparent bg, `primary` border (2px) + text, hover: bg `primary/5`
- **Social Button:** white bg, `border` border (1.5px), `rounded-xl`, icon + label, hover: bg `input-bg`
- **Input:**
  - At rest: bg `input-bg`, border `border` (1.5px), `rounded-xl`
  - Focus: border `primary` (1.5px), ring `primary/20`
  - Error: border `red-500`, error message in red below field
- **Glass Card:** a plain `div` using React inline `style` prop: `{{ background: 'rgba(255,250,246,0.92)', backdropFilter: 'blur(20px)', WebkitBackdropFilter: 'blur(20px)', border: '1px solid rgba(255,255,255,0.5)', boxShadow: '0 8px 40px rgba(0,0,0,0.18)', borderRadius: '20px' }}`. Do not use shadcn/ui `Card` — it does not support backdrop-blur natively. The `WebkitBackdropFilter` key is required for Safari/iOS support.

## Login Screen (`/login`)

### Layout

Full-screen viewport (`min-h-screen`) with warm gradient background and dark overlay. Contents:
- App logo fixed top-left (absolute)
- Tagline fixed bottom-left (absolute, hidden on mobile)
- Glass card centered (`flex items-center justify-center`)

### Card Dimensions

- **Desktop:** `max-w-[400px] w-full`, padding `p-9` (36px)
- **Mobile (< 640px):** `w-full`, `rounded-none`, `min-h-screen`, padding `pt-20 px-6 pb-8` — card fills the screen

### Card Contents (top to bottom)

1. **Heading:** "Welcome back" — Heading L
2. **Subheading:** "Sign in to continue planning your adventures" — Small, Muted
3. **Social buttons row (side by side):**
   - Google: `FcGoogle` icon from `react-icons/fc`, label "Continue with Google"
   - Apple: `FaApple` icon from `react-icons/fa`, label "Continue with Apple"
   - Both: Social Button style, `flex-1`
4. **Divider:** horizontal lines + "or continue with email" — Small, Muted
5. **Email field:** label "Email address", `type="email"`, placeholder "you@example.com"
6. **Password field:** label "Password", `type="password"`, placeholder "••••••••"
7. **"Forgot password?" link:** right-aligned, color `primary`, `href="#"` (non-functional in Phase 1)
8. **"Sign In" button:** Primary Button, full width
9. **"Don't have an account? Create one" link:** centered, `href="#"` (non-functional in Phase 1)

### Form Behavior (Phase 1 — UI only)

- **Submission:** clicking "Sign In" with valid inputs shows a loading state for 1 second then resets. No API call. Loading state: button text replaced by `Loader2` icon from `lucide-react` (animated spin via `animate-spin`, `w-5 h-5`), button disabled. Reset: loading state clears and button re-enables with "Sign In" text. Fields retain their values; validation errors are not cleared.
- **Validation (client-side):**
  - Email field: required, must be valid email format. Show inline error below field if empty or invalid on submit attempt.
  - Password field: required. Show inline error if empty on submit attempt.
  - Error message style: `text-red-500 text-xs mt-1`
- **No auth logic**, no tokens, no redirects in Phase 1.
